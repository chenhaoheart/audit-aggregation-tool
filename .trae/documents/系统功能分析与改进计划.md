# 空间数据检查桌面版 — 系统功能分析与改进计划

## 一、系统现状概览

本项目是基于 PySide6 的桌面应用，用于青海省山洪灾害风险隐患调查数据的质量检查、报表校验和 SHP 图层属性格式化。系统包含 3 个主页面（空间数据检查、成果报表展示、SHP格式化）、5 种主题、多线程检查服务、ArcGIS 集成等功能。

---

## 二、发现的问题与改进建议

### P0 — 紧急（影响正确性/稳定性）

#### 1. DEFAULT_RULES 浅拷贝导致默认值被污染
- **文件**: `core/config_manager.py` L164, L193
- **问题**: `DEFAULT_RULES.copy()` 是浅拷贝，内部字典仍为引用。修改 `self._rules` 中的字典会污染全局默认值，导致 `reset_to_default()` 无法恢复
- **修复**: 使用 `copy.deepcopy(DEFAULT_RULES)`

#### 2. batch_file_renamer 无错误处理，可能导致数据丢失
- **文件**: `core/batch_file_renamer.py` L177-189
- **问题**: `os.rename()` 无 try/except，文件占用或权限不足时崩溃；无重名检查，可能覆盖文件；无撤销机制
- **修复**: 添加 try/except、重名检查、预览功能

#### 3. AnimationPresets.fade_in else 分支控件不可见
- **文件**: `core/effects_manager.py` L306-316
- **问题**: else 分支只设了 `opacity: 0` 但没实现淡入动画，控件会保持不可见
- **修复**: 实现完整的淡入动画或使用 QGraphicsOpacityEffect

#### 4. 同名类 FieldMappingDialog 存在于两个文件
- **文件**: `ui/report_page.py` L2560 和 `ui/field_mapping_dialog.py` L828
- **问题**: 两个完全不同的实现但类名相同，极易造成导入错误
- **修复**: 重命名为 `ValidationFieldMappingDialog` 和 `ShpFormatterFieldMappingDialog`

#### 5. 关键操作缺失异常处理
- **文件**: `ui/report_page.py` L429, L887, L1730
- **问题**: `load_data()`、`run_data_check()`、`start_validation()` 无 try/except，异常导致应用崩溃
- **修复**: 添加 try/except + 用户友好的错误提示

#### 6. 裸 except 吞没所有异常
- **文件**: `core/data_validator.py` L203, L210; `ui/field_mapping_dialog.py` L183; `ui/arcgis_config_dialog.py` L83
- **问题**: `except:` 会捕获 `KeyboardInterrupt`、`SystemExit`，且无日志记录
- **修复**: 改为 `except Exception` 并添加日志

### P1 — 高（影响可维护性/扩展性）

#### 7. 编码检测逻辑三处重复且实现不一致
- **文件**: `core/checker.py` L84-152, `core/data_validator.py` L193-211
- **问题**: checker.py 有 LDID 检测 + 启发式选择，data_validator.py 只简单遍历编码列表，同一文件可能读取结果不同
- **修复**: 提取为 `core/encoding_utils.py` 公共模块

#### 8. 字段映射体系分裂，三套互不兼容
- **文件**: `config_manager.py`（`{输出字段: [源字段列表]}`）、`data_validator.py`（`{shp字段: 附表字段}`）、`checker.py`（硬编码候选列表）
- **问题**: 用户修改配置后可能不会在所有模块生效
- **修复**: 统一映射格式，所有模块通过 config_manager 获取映射

#### 9. report_page.py 过长（2968行，3个类）
- **问题**: 严重违反单一职责原则，可维护性极差
- **修复**: 拆分为 `report_page.py`、`validation_dialog.py`、`field_mapping_dialog_for_validation.py`

#### 10. 附表 Tab 设置方法三重复制
- **文件**: `ui/report_page.py` L215-327
- **问题**: `setup_fubiao1/2/3_tab()` 代码几乎完全相同，筛选/清除方法也三重复制
- **修复**: 提取为通用方法 `_create_fubiao_tab(name, search_placeholder)`

#### 11. 表头定义多处重复
- **文件**: `ui/report_page.py` L474-517, L717-760, L994-1031
- **问题**: `header_row1/2/3`、`data_headers` 被复制了3次，约360行重复代码
- **修复**: 提取为类常量或模块级常量

#### 12. 导出方法大量重复
- **文件**: `ui/report_page.py` L690-879, L939-1220
- **问题**: `export_excel()` 和 `export_check_report()` 共享约70%代码
- **修复**: 提取公共导出逻辑（创建Workbook、设置样式、写入数据、合并单元格）

#### 13. checker.py 三个校验方法大量重复
- **文件**: `core/checker.py`
- **问题**: `validate_duanmian/fangzhi/yinhuan` 中河流代码校验逻辑几乎完全相同
- **修复**: 提取公共的 `_validate_river_code()` 方法

#### 14. load_fubiao2/3 完全重复
- **文件**: `core/report_reader.py` L168-229
- **问题**: 两个方法代码完全一样，仅函数名不同
- **修复**: 提取为通用 `_load_simple_fubiao(file_path, name_keywords)` 方法

#### 15. effects_manager.py 单文件13个类
- **文件**: `core/effects_manager.py`（883行）
- **问题**: 严重违反单一职责原则
- **修复**: 拆分为 `animations.py`、`shadows.py`、`buttons.py`、`transitions.py` 等

#### 16. batch_file_renamer 未接入主题系统
- **文件**: `core/batch_file_renamer.py`
- **问题**: 所有 UI 样式硬编码，暗色主题下浅色文字不可见
- **修复**: 使用 ThemeManager 管理样式

### P2 — 中（影响一致性/健壮性）

#### 17. 异常处理策略不一致
- **问题**: 有的 `except Exception: pass`，有的 `except: continue`，有的 `print()`，有的无处理
- **修复**: 统一为 `logging` 模块，定义错误处理规范

#### 18. ThemeManager 单例模式实现有缺陷
- **文件**: `core/theme_manager.py` L71-78
- **问题**: `__new__` + `__init__` 的单例模式在异常时不一致，QObject 不应覆盖 `__new__`
- **修复**: 使用模块级 `get_theme_manager()` 函数代替 `__new__` 单例

#### 19. _get_field_value 模糊匹配过于宽松
- **文件**: `core/data_validator.py` L478-480
- **问题**: `field_name in k or k in field_name` 导致"名称"匹配到"5.名称"、"11.名称"等多个字段
- **修复**: 使用更精确的匹配策略（如完全匹配优先，再尝试后缀匹配）

#### 20. 硬编码颜色未使用主题系统
- **文件**: `ui/report_page.py` L526, L916, L1989; `ui/pages/check_page.py` L577-583
- **问题**: 多处使用 `QColor('#xxx')` 硬编码颜色，暗色主题下可能不可见
- **修复**: 通过 ThemeManager 获取主题色

#### 21. 入场动画代码5处复制
- **文件**: report_page.py, arcgis_config_dialog.py, theme_dialog.py, log_dialog.py
- **问题**: `showEvent` + `_animate_entrance` 代码完全相同
- **修复**: 提取为 `FadeInMixin` 混入类

#### 22. 主题调色板设置代码7处重复
- **问题**: `theme_manager.get_current_theme()` + `setPalette()` + `setAutoFillBackground()` 代码块重复
- **修复**: 提取为 `apply_theme_palette(widget)` 工具函数

#### 23. Excel 读取两套实现
- **文件**: `report_reader.py`（pandas + openpyxl）和 `attachment_reader.py`（纯 openpyxl）
- **问题**: 合并单元格处理方式不同，同一文件可能解析结果不一致
- **修复**: 统一为一套实现，或明确各自的使用场景

#### 24. config_manager import_config 验证不充分
- **文件**: `core/config_manager.py` L228-232
- **问题**: 仅验证顶层键存在，不验证值类型和内容
- **修复**: 增加类型和内容验证

#### 25. shp 文件名硬编码
- **文件**: `core/data_validator.py` L247, L261
- **问题**: `if file == '防治对象分布P.shp'` 完全硬编码
- **修复**: 使用配置或关键词匹配

### P3 — 低（影响规范性/体验）

#### 26. 方法命名不一致
- **问题**: `init_ui()` vs `_init_ui()`，`finished_signal` vs `check_finished`
- **修复**: 统一为 `_init_ui()`（私有方法风格），信号统一为 `xxx_changed`/`xxx_finished`

#### 27. 导入风格不一致
- **问题**: 有的顶部导入，有的方法内延迟导入
- **修复**: 重模块顶部导入，可选依赖（如 pandas、openpyxl）在 try 块内导入

#### 28. 校验时总是弹出字段映射对话框
- **文件**: `ui/report_page.py` L1748
- **问题**: 每次校验都弹出 FieldMappingDialog，即使用户已配置
- **修复**: 提供"使用上次配置"选项

#### 29. 加载数据时无进度指示
- **文件**: `ui/report_page.py` L429
- **问题**: `load_all_reports()` 在主线程同步执行，界面冻结
- **修复**: 使用 QThread 或 QRunnable 异步加载

#### 30. check_page.py 空回调方法
- **文件**: `ui/pages/check_page.py` L599
- **问题**: `_on_state_changed()` 方法体只有 `pass`
- **修复**: 删除或实现

#### 31. check_page.py 调用私有方法
- **文件**: `ui/pages/check_page.py` L888-892
- **问题**: 直接调用 `FilterBar._clear_filter()` 私有方法
- **修复**: 通过公共接口（`clear_requested` 信号）实现

#### 32. 可访问性问题
- **问题**: 无键盘快捷键、仅依赖颜色区分状态、QLabel 徽章无障碍描述缺失
- **修复**: 添加快捷键、图标辅助、accessibleName
