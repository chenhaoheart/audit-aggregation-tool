# 风险隐患调查与影响分析成果审核小工具 - PRD文档

## 1. 产品概述

### 1.1 产品名称
风险隐患调查与影响分析成果审核小工具（青海空间数据检查工具）

### 1.2 产品定位
面向青海省山洪灾害风险隐患调查与影响分析项目成果审核的专用桌面工具，为审核人员提供空间数据质量检查、成果报表展示和SHP属性格式化等功能，提高成果审核效率和准确性。

### 1.3 目标用户
- 青海省水利厅/应急管理部门审核人员
- 区县级山洪灾害调查成果审核工作人员
- 空间数据质量控制技术人员

### 1.4 产品价值
- **提升审核效率**：自动化检查替代人工逐一核对，大幅减少审核时间
- **降低错误率**：规则化检查避免遗漏，确保数据完整性
- **标准化输出**：统一命名和格式规范，便于成果归档和验收

## 3. 功能详细设计

### 3.1 空间数据检查模块

#### 3.1.1 功能描述
对青海省山洪灾害调查成果中的空间数据（SHP图层）进行自动化检查，验证数据完整性和一致性。

#### 3.1.2 输入要求
| 输入项 | 类型 | 说明 |
|--------|------|------|
| 目标文件夹 | 目录 | 包含多个流域/子文件夹的根目录 |
| 水系文件 | SHP文件 | 水系基础数据文件，包含河流代码(RVCD)和河流名称(RVNM) |

#### 3.1.3 检查对象
系统自动递归搜索目标文件夹，检查以下SHP图层：
- **断面平面位置L.shp** - 河流断面空间位置数据
- **防治对象分布P.shp** - 山洪灾害防治对象分布数据
- **隐患要素分布L.shp** - 隐患要素空间分布数据

#### 3.1.4 检查规则详解

##### A. 水系图层检查规则
| 检查项 | 规则说明 | 错误提示 |
|--------|----------|----------|
| 河流代码长度 | 必须为17位 | "河流代码[xxx]长度xx位(应为17位)" |
| 河流代码重复 | 同一河流代码不能重复出现 | "河流代码[xxx]重复出现N次" |

##### B. 断面平面位置L.shp检查规则
| 检查项 | 规则说明 | 验证字段 |
|--------|----------|----------|
| 河流代码长度 | 必须为17位 | 河流代码 |
| 河流代码存在性 | 必须在水系数据中存在 | 河流代码 vs 水系RVCD |
| 河流名称一致性 | 河流代码对应的名称必须与水系一致 | 河流名称 vs 水系RVNM |
| 编号长度 | 必须为23位 | 编号 |
| 编号前缀匹配 | 编号前17位必须与河流代码一致 | 编号[:17] vs 河流代码 |
| 编号后缀匹配 | 编号后5位必须与断面名称后5位一致 | 编号[-5:] vs 名称[-5:] |
| 断面名称格式 | 必须符合"河流名称+CS+3位数字"格式 | 名称正则匹配 |

##### C. 防治对象分布P.shp检查规则
| 检查项 | 规则说明 | 验证字段 |
|--------|----------|----------|
| 河流代码长度 | 必须为17位 | 河流代码 |
| 河流代码存在性 | 必须在水系数据中存在 | 河流代码 vs 水系RVCD |
| 河流名称一致性 | 河流代码对应的名称必须与水系一致 | 河流名称 vs 水系RVNM |
| 代码唯一性 | 代码字段不能重复 | 代码 |

##### D. 隐患要素分布L.shp检查规则
| 检查项 | 规则说明 | 验证字段 |
|--------|----------|----------|
| 河流代码长度 | 必须为17位 | 河流代码 |
| 河流代码存在性 | 必须在水系数据中存在 | 河流代码 vs 水系RVCD |
| 河流名称一致性 | 河流代码对应的名称必须与水系一致 | 河流名称 vs 水系RVNM |
| 编号长度 | 必须为28位 | 编号 |
| 编号政区代码 | 编号开头6位必须是数字（政区代码） | 编号[:6] |
| 编号代码匹配 | 编号7-23位必须与河流代码一致 | 编号[6:23] vs 河流代码 |
| 编号唯一性 | 编号字段不能重复 | 编号 |

#### 3.1.5 输出结果
检查完成后，以Tab页形式展示以下结果：

| Tab页 | 内容 |
|-------|------|
| 汇总统计 | 各文件检查状态、总记录数、有效/无效记录数、有效率 |
| 断面平面位置 | 每条记录的详细检查结果，包含所有验证字段 |
| 防治对象分布 | 每条记录的详细检查结果 |
| 隐患要素分布 | 每条记录的详细检查结果 |
| 水系数据 | 水系基础数据的检查结果 |

#### 3.1.6 筛选功能
每个数据Tab页支持筛选：
- **状态筛选**：下拉选择"全部/通过/不通过"
- **代码筛选**：输入框筛选河流代码
- **名称筛选**：输入框筛选河流名称（水系Tab页除外）

#### 3.1.7 导出功能
检查结果可导出为Excel文件（检查结果.xlsx），包含：
- 水系数据概览Sheet
- 各文件明细Sheet（按源文件分Sheet）
- 汇总统计Sheet

### 3.2 成果报表展示模块

#### 3.2.1 功能描述
读取并展示青海省山洪灾害调查成果附表数据，并与空间数据进行双向匹配验证。

#### 3.2.2 附表数据结构

| 附表 | 名称 | 主要字段 |
|------|------|----------|
| 附表1 | 山洪灾害防治对象名录 | 县名、乡镇名、名称、代码、类型、人口、河流名称、河流代码等 |
| 附表2 | 跨沟道路、桥涵、塘（堰）坝调查成果表 | 名称、编码、类型、河流名称、河流代码等 |
| 附表3 | 沟滩占地情况调查成果表 | 名称、编码、类型、河流名称、河流代码等 |

#### 3.2.3 字段映射配置
系统支持配置附表字段与SHP字段的映射关系，默认映射：

```
附表1 ↔ 防治对象分布P.shp：
  - 名称 ↔ 名称
  - 代码 ↔ 代码

附表2 ↔ 隐患要素分布L.shp：
  - 名称 ↔ 名称
  - 编号 ↔ 编码

附表3 ↔ 隐患要素分布L.shp：
  - 名称 ↔ 名称
  - 编号 ↔ 编码
```

#### 3.2.4 双向匹配验证
| 匹配方向 | 检查内容 |
|----------|----------|
| 附表 → SHP | 附表数据是否都在空间数据中存在（缺失空间位置） |
| SHP → 附表 | 空间数据是否都在附表中存在（缺失登记信息） |

### 3.3 SHP属性格式化模块

#### 3.3.1 功能描述
批量处理流域文件夹中的SHP文件，统一命名和字段格式，输出规范化的成果文件。

#### 3.3.2 环境依赖
- **ArcGIS Pro** 或 **ArcGIS Desktop 10.8**
- 需配置ArcGIS Python环境路径

#### 3.3.3 处理流程
```
输入目录（流域根目录）
    ├── 流域A/
    │   ├── 断面平面位置L.shp (原始命名可能不一致)
    │   ├── 防治对象分布P.shp
    │   └── 隐患要素分布L.shp
    └── 流域B/
        └── ...

处理 → 字段重映射 + 文件重命名

输出目录
    ├── 流域A/
    │   ├── 断面平面位置L.shp (统一命名)
    │   ├── 防治对象分布P.shp
    │   └── 隐患要素分布L.shp
    └── 流域B/
        └── ...
```

#### 3.3.4 字段映射规则（rules_config.json）
系统通过关键词匹配识别文件类型，按配置进行字段映射：

| 输出文件 | 匹配关键词 | 字段映射 |
|----------|------------|----------|
| 断面平面位置L.shp | "断面" | 名称→断面NM/断面/名称, 编号→PID/编号, 河流名称→RVNM, 河流代码→RVCD |
| 防治对象分布P.shp | "防治对象/保护对象" | 名称→NAME/名称, 代码→PID/代码, 河流名称→RVNM, 河流代码→RVCD |
| 隐患要素分布L.shp | "隐患要素" | 名称→NAME/名称, 编号→PID/编号, 河流名称→RVNM, 河流代码→RVCD |

#### 3.3.5 处理结果展示
结果表格显示：
- 序号
- 流域/目录名称
- 输出文件路径
- 记录数
- 状态（成功/失败）

### 5.3 核心类设计

#### WaterSystemChecker（checker.py）
```
class WaterSystemChecker:
    属性:
        - folder_path: 目标文件夹
        - water_system_shp: 水系文件路径
        - water_codes: 河流代码集合
        - water_names: 河流名称集合
        - water_code_to_name: 代码→名称映射
        - results: 检查结果列表
        - progress_callback: 进度回调函数

    方法:
        - load_water_system(): 加载水系数据
        - validate_duanmian(row): 断面数据校验
        - validate_fangzhi(row): 防治对象校验
        - validate_yinhuan(row): 隐患要素校验
        - process_all(): 执行全部检查
```

#### DataValidator（data_validator.py）
```
class DataValidator:
    属性:
        - fubiao_data: 附表数据

    方法:
        - load_fubiao(report_data): 加载附表数据
        - load_shp_data(folder_path): 加载空间数据
        - validate_fubiao1_vs_fangzhi(): 附表1与防治对象对比
        - validate_fubiao23_vs_yinhuan(): 附表2/3与隐患要素对比
        - validate_all(): 执行全部校验
```

#### ThemeManager（theme_manager.py）

class ThemeManager:

### 5.4 数据编码处理
系统采用多种策略处理SHP文件中文编码问题：
1. **LDID检测**：读取DBF文件头部的Language Driver ID
2. **启发式检测**：尝试多种编码（UTF-8、GB18030、GBK、CP936）
3. **字段非空数验证**：选择能读取最多有效数据的编码
