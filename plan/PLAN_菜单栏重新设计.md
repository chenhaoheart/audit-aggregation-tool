# 菜单栏重新设计方案

## 一、现状分析

### 当前实现
项目中有三个菜单/导航相关的组件：

| 组件 | 文件路径 | 风格 | 状态 |
|------|---------|------|------|
| AntSidebar | `ui/components/ant_menu.py` | macOS Finder 风格，支持多级展开 | 主用 |
| Sidebar | `ui/components/sidebar.py` | 简洁导航按钮 | 未使用 |
| DockBar | `ui/components/dock_bar.py` | macOS Dock 风格，大图标 | 未使用 |

### 当前菜单结构
```
🌏 审核汇集
  风险隐患调查审核工具
  ─────────────────────
  🔍 数据检查
    ├── 检查配置
    ├── 结果总览
    └── 数据明细
        ├── 断面平面位置
        ├── 防治对象分布
        ├── 隐患要素分布
        └── 水系数据
  📊 成果报表展示
  🛠️ SHP属性格式化
  ─────────────────────
  ◀ 折叠        ⚙️
```

## 二、重新设计方案

### 方案A：现代化 Finder 风格（推荐）

保持现有的 macOS Finder 风格，优化细节：

**改进点：**
1. 更精致的视觉层次
2. 平滑的折叠/展开动画
3. 更好的选中状态指示
4. 图标与文字对齐优化

**预期效果：**
- 展开宽度：220px
- 折叠宽度：72px
- 动画时长：250ms

### 方案B：水平标签导航

将侧边栏改为顶部水平标签栏：

**布局：**
```
┌─────────────────────────────────────────────────┐
│  🌏 审核汇集              主题切换 │ 设置 │ 帮助 │
├─────────────────────────────────────────────────┤
│  [空间数据检查] [成果报表展示] [SHP属性格式化]   │
├─────────────────────────────────────────────────┤
│                                                 │
│                  内容区域                        │
│                                                 │
└─────────────────────────────────────────────────┘
```

**优点：**
- 更开阔的内容区域
- 适合宽屏显示器
- 现代化 UI 风格

**缺点：**
- 占用垂直空间
- 不支持折叠

### 方案C：混合导航模式

结合侧边栏 + 顶部工具栏：

**布局：**
```
┌─────────────────────────────────────────────────┐
│  🌏 风险隐患调查审核工具              ⚙️ 主题 ▼  │
├──────────┬──────────────────────────────────────┤
│ 🔍 数据检查│                                       │
│   ├ 配置  │                                       │
│   ├ 总览  │              内容区域                 │
│   └ 明细  │                                       │
│ 📊 报表   │                                       │
│ 🛠️ 格式化│                                       │
├──────────┴──────────────────────────────────────┤
│  状态栏：就绪                    v1.3.0          │
└─────────────────────────────────────────────────┘
```

1. 菜单栏在所有5种主题下显示正确
2. 折叠/展开动画流畅（60fps）
3. 点击响应时间 < 100ms
4. 支持键盘导航（Tab/方向键）
5. 与现有 CheckPage、ReportPage、ShpFormatterPage 集成正常
